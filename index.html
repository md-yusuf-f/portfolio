01 TCS · 2023

Event-Driven Kafka Transaction Pipeline

Problem
A 3-hop synchronous REST chain processed each financial transaction sequentially. Any single-hop failure caused a full-chain failure requiring manual retry. Under month-end batch load, P99 latency hit 850ms and consumer timeouts became routine.
Solution
Replaced the blocking chain with an async Kafka pipeline — partitioned consumers, dead-letter queue with retry handling, and idempotent settlement writes.
Result
P99 latency reduced from 850ms to 550ms. Peak throughput of ~1,200 msg/sec. Horizontal scale by adding consumers — no code changes required.
