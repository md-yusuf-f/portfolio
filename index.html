04 TCS · 2023

Event-Driven Kafka Pipeline

Replaced synchronous REST calls with an event-driven Apache Kafka pipeline for high-throughput financial transaction processing at TCS. Designed producer/consumer topology with partitioned topics to eliminate blocking I/O bottlenecks, reducing end-to-end latency by 35% and improving system throughput by 40%.

# Before: synchronous REST (blocking) Service A ──[HTTP POST]──> Service B ──[HTTP POST]──> Service C # After: async Kafka (non-blocking) Service A ──[Produce]──> Topic: txn.events Service B ──[Consume]── Topic: txn.events (parallel consumers) Service C ──[Consume]── Topic: txn.processed # Result: 35% latency ↓, 40% throughput ↑
